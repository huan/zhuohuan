关于我的“老烂站”的一些特性（包括实现的和没实现的：）

PreLoad：我打算划出一个没有大小的 Frame， 然后当 Client 端读某篇文章的时候，用 JavaScript 同时将我猜测 Client 将要阅读的下篇文章的 URL 写入那个没有大小的 Frame ，耩子如果 Client 的阅读顺序被我猜中，访问速度将大大提高。[未实现]
Daemons：也是存在于一个没有大小的 Frame 中。将所有常用的 JavaScript 函数都写在这个 Frame 中，别的 Frame 需要的时候可以直接用 Parent.Daemons.Function() 。防止每个 HTML 文件内都要放上相同的 JavaScript 原码。[实现]
Chatting：在左上角的那个大大的眼睛上做文章：每个 OnMouseOver 函数触发一次聊天信息（用 Prompt() 来实现和客户交互：给出回答并允许客户输入）。因为在 Mud 中有“李凌”的例子，所以应该不难。[未实现]
