在我的感觉中，Java 所谓的一次编译，到处运行的能力和 Basic 语言是一个特性（它只不过把编程和编译这两个概念给混淆视听了……Basic是直接解释原码，而JVM 是解释处理过的原码……）。在 Basic 语言中，不管是在 Apple 还是 PC 上，只要运行了解释器，就能运行。而 Java 比 Basic 先进的地方无非是面向对象的特性、强壮的特性、强大的网络支持，安全支持的特性罢了。而 Java 语言这些强大的功能运行在一个解释器 JVM（我觉得极有讽刺意味的是：无论怎样，JVM 也是和硬件平台软件平台有关的）中，效率底的要命。而 C++ 语言虽然不能一次编程到处运行，但是却可以一次编程，到处编译，而且速度极快。所以我对 Java 的前景并不抱乐观的态度。

对于 Java 的软件系统：

HotJava浏览器，比“硬件杀手”IE 的“杀伤力”还要强，而且要是遇到网页内有大量的 JavaScript 与 JavaApplet 的网页，在 Pentium200，16M 的机器上都没有办法用……（速度太慢了！）
其他正在开发和已经开发出来的 Java 大系统，应该说没有必要为了什么一次编译到处运行而牺牲巨大的性能……因为毕竟大系统的运行平台是稳定的，不会换来换去，就算需要运行在不同的平台上，也可以用 C++ 编译两次。

对于网页中的 JavaApplet:

效率问题和 Java 一样，区别是应该说不是非常严重：毕竟 Applet 都非常小。但是有两个问题：
用大量金钱换取的网络传输速度毕竟有限，一个 Applet 怎么也要十几 K ，需要考虑值不值得为了一个网页花花绿绿的效果而使用酱紫大的一个 Applet。
每当 Applet 启动的时候，在中档机器上（Pentium 166）的浏览器都会有很长一段时间好象死机了似的，喀嚓硬盘，浪费时间。
所以我认为不如把 Applet 换成 JavaScript 或 DHTML 技术，它们都非常小，而效果也非常好。

对于网页中的 JavaScript:

用不大的程序就能实现一些很强的功能，现在又出现的 DHTML(Dynamic HTML：动态HTML） 技术也是建立在 JavaScript 基础上的，所以我认为最有前途。^_^
MicroSoft 公司为了显示自己，弄了 VBScript，竟然在 Visual C++ 的连机帮助中都有 VBScript 的帮助……霸道至极。不过熟悉 Visual Basic 的大虾好象方便了吧……

个人感觉，呵呵。
